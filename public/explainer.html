The problem

A smooth model over un-smooth terrain

A global model resolves the land as gentle slopes, so it paints broad, uniform streamlines. Real ridges split, funnel, and reverse the wind. The animated layer can even imply air moving a direction no nearby vane supports — which is exactly what makes a fire run.

Where it bites

On a ridge, a vane can read a hard cross-slope wind while the model a few hundred metres away shows a calm, on-shore drift. Same map, opposite stories. The model’s confidence is uniform; the terrain’s behaviour is not.

…and the naïve “fix” is worse

Reproducing the vanes from airport METARs only samples the flats that ring the range — the most model-agreeable spots — so it hides the disagreement instead of showing it. The real vanes sit on the terrain.

Auditing the first build surfaced four logic flaws that all pointed the same way — they quietly understated the conflict:

  • Wrong source. Wind came from ~10 airport METARs, not Watch Duty’s real provider (Synoptic: RAWS, CWOP/personal, mesonet).
  • Calm vanes dropped. A calm station reports a null direction; the code threw it away — so vanes vanished on the lightest-wind days.
  • The field never touched the vanes. A normalization bug (÷ max(Σw,1)) meant the “vane-corrected” field never actually reached the observation it was pinned to.
  • The disagreement view ran backwards. It animated the difference vector, whose direction is noise when model and vanes roughly agree — so flow appeared to run the wrong way for no reason.
The proposed solution

Pin the wind field to the real vanes — and show the gap honestly

Take the model as a first guess, then nudge it toward every real vane using Barnes objective analysis: interpolate each vane’s residual (observation − model) with a distance-weighted Gaussian and add it back. The field bends to the vanes where there’s ground truth and relaxes to the model where there isn’t.

corrected(x) = model(x) + Σ wᵢ·rᵢ ⁄ (Σ wᵢ + k)   where   wᵢ = e−dᵢ²/R²,   rᵢ = obsᵢ − model
The + k acts like one far-away “the model is probably right here” observation: near a vane (Σw ≫ k) the field honours it; in the empty interior (Σw → 0) it fades back to the model.

Real source, every vane

One Synoptic stations/latest query returns the whole network Watch Duty draws — RAWS on the ridges, CWOP sensors on rooftops, DOT mesonets in the valleys — instead of a handful of airports.

An honest disagreement view

Particles advect through the real (vane-corrected) wind, so direction is meaningful, and are coloured by the model↔vane gap — the map glows where the model is wrong rather than flowing in a made-up direction.

The takeaway

The model is a guess. The vanes are the measurement.

When a fire is moving, the right question isn’t “what does the model say” — it’s “what do the stations on this ridge say, and where does the pretty animation disagree with them.” This makes that gap visible.
