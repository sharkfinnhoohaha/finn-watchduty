The problem

A smooth global model over un-smooth terrain

Watch Duty draws two wind pictures on one map: a smooth animated model and real weather vanes (Synoptic — RAWS, CWOP, mesonet). In the mountains they frequently disagree — the model can't resolve ridge-channeled flow — yet the animation is shown as if it were ground truth. During a fire, that gap is a safety issue.

Where it bites

A ridge vane reads a hard cross-slope wind while the model a few hundred metres away shows a calm on-shore drift. Same map, opposite stories.

The naïve "fix" is worse

Reproducing the vanes from airport METARs only samples the flats that ring the range — the most model-agreeable spots — so it hides the gap instead of showing it.

The solution — three honest views

Pin the field to the vanes; show the gap

MODEL Smooth & uniform
One direction everywhere — can't see terrain.
VANE-CORRECTED Bent to truth
Arrows are real vanes; field pins to them.
DISAGREEMENT Flow = wind · glow = gap
Real flow; glows where the model is wrong.

The Disagreement view is the one to ship first: it never claims a derived field is truth — it just flags where the pretty animation disagrees with your own vanes.

How it works

Barnes objective analysis

Take the model as a first guess, then nudge it toward every vane: interpolate each vane's residual (observation − model) with a distance-weighted Gaussian and add it back.

corrected(x) = model(x) + Σ wᵢ·rᵢ ⁄ (Σ wᵢ + k)
  • Near a vane (Σw ≫ k): the field honours the observation.
  • In the empty interior (Σw → 0): it relaxes back to the model — no overconfident guesses.
  • Disagreement = advect particles through the real wind, colour by |corrected − model|.

Rollout — low-risk first

Phased, feature-flagged, reversible

Phase 0 · Spike

≈1–2 days. Port the TS module + overlay into a branch, fed by your Synoptic + model data, behind a flag. Validate on one fire.

Phase 1 · Disagreement overlay THE WEDGE

≈1 week. Ship only the glow-where-it-disagrees toggle. Doesn't touch the model layer; never claims truth. Lowest risk, highest trust.

Phase 2 · Corrected field

≈1–2 weeks. Add the vane-corrected flow + radius control + per-station model↔vane readout.

Phase 3 · Terrain + alerting

Elevation-aware weighting (RAWS elevation is in Synoptic) so ridge wind doesn't bleed into canyons; "alert me when the model and ridge vanes diverge by > X" on your existing alert system.

Throughout

Additive & reversible. Reuses Synoptic + Mapbox + your Python backend. Pure TS/Python — the languages you onboard volunteers on.
